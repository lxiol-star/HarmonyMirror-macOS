# 网络连接优化说明

## 修复的问题

### 1. IP 地址格式验证和规范化
**问题**: 用户输入的 IP 地址格式错误(如 `10.21.50.4.37:37669`)导致连接失败
**修复**: 
- 在 `HDCCommand.wifiTarget()` 中添加 IP 格式验证和规范化
- 新增 `normalizeIPv4()` 方法自动修正常见的格式错误
- 在连接前验证 IP 格式,提前给出明确的错误提示

### 2. 网络切换后自动重连
**问题**: Mac 切换网络后,之前连接的 WiFi 设备丢失,需要手动重新连接
**修复**:
- 在 `DeviceDiscovery.poll()` 中添加自动重连逻辑
- 检测已记住的 WiFi 设备是否丢失,自动尝试重连
- 使用失败冷却机制避免频繁重试失败的连接

### 3. 改进的错误提示
**问题**: 连接失败时错误信息不够明确
**修复**:
- 在 `DeviceListView.connectWiFi()` 中添加详细的错误分类
- 针对不同错误类型给出具体的解决建议:
  - IP 格式错误 → 提示检查输入
  - 连接超时 → 提示检查网络
  - 连接被拒绝 → 提示开启无线调试

### 4. 网络诊断功能
**已有功能**: `NetworkDiagnostics` 提供完整的网络诊断
- 检测本机 IP 和网关
- 探测目标设备端口是否可达
- 测量网络延迟
- 给出针对性的建议

## 使用说明

### 手动连接 WiFi 设备
1. 在输入框中输入设备 IP(支持多种格式):
   - `10.21.50.4` (自动添加默认端口)
   - `10.21.50.4:37669` (指定端口)
   - 即使输入格式有误(如 `10.21.50.4.37:37669`),也会自动修正

2. 点击"连接 Wi-Fi"按钮

3. 如果连接失败,查看错误提示:
   - "IP 地址格式错误" → 检查输入的 IP 是否正确
   - "连接超时" → 确保设备与 Mac 在同一网络
   - "连接被拒绝" → 在设备上开启无线调试

### 网络诊断
点击"网络诊断"按钮(网络图标),可以查看:
- 本机 IP 地址
- 网关地址
- 目标设备端口是否可达
- 网络延迟
- 针对性的建议

### 自动重连
- 应用会记住成功连接过的 WiFi 设备
- 网络切换后,应用会自动尝试重连这些设备
- 无需手动重新输入 IP 地址

## 技术细节

### IP 格式规范化算法
```swift
// 处理多余的点号和格式错误
"10.21.50.4.37:37669" → "10.21.50.4:37669"
"10.21.50.4" → "10.21.50.4:10178"
```

### 自动重连策略
- 每次轮询时检查已记住的设备是否在线
- 失败的连接会有 30 秒冷却期
- 避免频繁重试导致性能问题

### 错误分类
- IP 格式错误: 在本地验证,不发送到 hdc
- 连接超时: 网络不可达或设备离线
- 连接被拒绝: 设备未开启无线调试端口

## 测试建议

1. **格式容错测试**:
   - 输入 `10.21.50.4.37:37669` 应该自动修正为 `10.21.50.4:37669`
   - 输入 `192.168.1.100` 应该自动添加端口 `:10178`

2. **网络切换测试**:
   - 连接 WiFi 设备后,切换 Mac 的网络
   - 等待几秒,应该自动重连

3. **错误提示测试**:
   - 输入错误的 IP → 应该提示"IP 地址格式错误"
   - 输入不可达的 IP → 应该提示"连接超时"
   - 输入未开启调试的设备 IP → 应该提示"连接被拒绝"

4. **网络诊断测试**:
   - 点击网络诊断按钮
   - 应该显示本机 IP、目标状态和建议
